Man vajag, lai tu soli pa solim pārbaudi, kāpēc produkti no Sanity neparādās mājaslapā, lai gan Sanity panelī tie ir ievadīti.

Mērķis:
Panākt, lai, kad klients nomaina vai pievieno produktu Sanity panelī, tas automātiski parādās mājaslapā.

Lūdzu pārbaudi šīs lietas PRECĪZI un salabo:

Vai frontend tiešām pieslēdzas pareizajam Sanity projektam:
projectId
dataset
apiVersion
useCdn
vai tiek izmantots pareizais client config
Vai .env mainīgie ir pareizi nolasīti:
SANITY_PROJECT_ID
SANITY_DATASET
SANITY_API_VERSION
SANITY_TOKEN, ja vajag
vai mainīgo nosaukumi sakrīt ar to, ko izmanto kodā
Vai query uz Sanity ir pareizs:
pārbaudi GROQ query
vai query vispār atgriež produktus
izdrukā console log vai response satur produktus
ja vajag, uztaisi minimālu test query tikai produktiem
Vai Sanity dokumentu struktūra sakrīt ar frontend gaidīto struktūru:
piemēram: title, slug, price, image, description
ja frontend gaida citus field nosaukumus nekā Sanity dokumentos, izlabo mapping
Vai tiek pareizi apstrādāti attēli:
vai image field ir asset
vai tiek lietots @sanity/image-url, ja vajag
vai broken image URL neapstādina visu produktu renderēšanu
Vai produkti netiek noslēpti frontend pusē:
filter
draft/published statuss
category filtrs
tikai featured produkti
tukšs array state
loading/error state
Vai lapā ir pareizs data fetching:
ja Next.js, pārbaudi server component / client component loģiku
vai fetch notiek build time, request time vai client side
ja statiska lapa, vai vajag revalidate
ja vajag, pievieno revalidate vai dynamic fetch
Vai Sanity dokumenti ir publicēti, nevis tikai draft:
ja query neiekļauj draftus, pārbaudi published versijas
Pievieno debug informāciju:
console log saņemtajiem datiem
console log query rezultātam
console log produktu skaitam
ja ir error, parādi precīzu error tekstu
Man vajag gala rezultātu:
atrast konkrēto iemeslu, kāpēc produkti nerādās
salabot kodu
īsi paskaidrot, kas bija nepareizi
uzrakstīt, kuri faili tika mainīti
uzrakstīt, vai vēl no manis vajag kādu ID / key / env vērtību

Svarīgi:

Ja pietrūkst kāda Sanity vērtība, tad pasaki tieši kuru:
projectId
dataset
token
schema nosaukums
query
field nosaukumi
Nevis minēt, bet pārbaudīt ar reāliem logiem un koda analīzi.
