Integrate Sanity CMS into the existing product system.

GOAL:
Replace static/mock products with real data coming from Sanity.

SETUP:

Install Sanity client:
npm install @sanity/client @sanity/image-url
Create a Sanity client file:
/lib/sanity.ts

Example:

import {createClient} from '@sanity/client'

export const client = createClient({
  projectId: 'vsh21wfq',
  dataset: 'production',
  apiVersion: '2024-01-01',
  useCdn: true,
})

FETCH PRODUCTS:

Create function:

export async function getProducts() {
  return await client.fetch(`
    *[_type == "product"]{
      _id,
      title,
      "slug": slug.current,
      mainImage,
      shortDescription,
      category,
      featured
    }
  `)
}

IMAGES:

Add helper:

import imageUrlBuilder from '@sanity/image-url'
import {client} from './sanity'

const builder = imageUrlBuilder(client)

export function urlFor(source) {
  return builder.image(source)
}

FRONTEND INTEGRATION:

Replace existing mock products with getProducts()
Load products in:
homepage
product listing page
Display:
title
image
description
category

PRODUCT PAGE:

Create dynamic route:

/products/[slug]

Fetch single product:

*[_type == "product" && slug.current == $slug][0]

FILTERING:

Allow filtering by:

category
featured

EXPECTED RESULT:

Products fully controlled from Sanity Studio
No hardcoded product data
Dynamic product pages working via slug
Images loaded from Sanity CDN

IMPORTANT:

Use async/await
Handle loading states
Handle empty data safely
Do not break existing layout
